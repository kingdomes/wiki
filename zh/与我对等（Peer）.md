# 与我进行 Peering

本页面提供有关与 **AFNOOI-Network** 进行对等连接（Peering）的入门信息。

---

## 概述

AFNOOI-Network 欢迎 DN42 社区成员建立对等互联。我们秉持开放、友好的互联政策，致力于构建稳定、低延迟的网络拓扑。

---

## 对等要求

在发起对等请求之前，请确认满足以下基本要求：

| 项目 | 要求 |
| :--- | :--- |
| **ASN** | 拥有有效的 DN42 ASN |
| **IP 地址段** | 拥有至少一个 IPv4 和/或 IPv6 地址段 |
| **隧道协议** | 支持 WireGuard 或 GRE 隧道 |
| **BGP 协议** | 支持 BGP（推荐使用 BIRD 或 FRRouting） |
| **网络稳定性** | 拥有相对稳定的网络连接，避免频繁断线 |

---

## 支持的隧道协议

| 协议 | 说明 |
| :--- | :--- |
| **WireGuard** | 推荐协议，配置简单，安全性高 |
| **GRE** | 支持标准 GRE 隧道 |

> *如果您需要其他协议，请提前沟通确认。*

## BGP 功能支持

AFNOOI-MNT 网络使用 **BIRD 2**，并支持以下特性：

| 特性 | 说明 | RFC |
| :--- | :--- | :--- |
| **Multiprotocol BGP** | 多协议 BGP | RFC 4760 |
| **BGP Large Communities** | BGP 大型社区 | RFC 8092 |
| **BGP Confederations** | BGP 联盟 | RFC 5065 |
| **Extended Next Hop** | 扩展下一跳 | RFC 5549 |
| **Extended Messages** | 扩展消息 | RFC 8654 |
| **dn42 ROA** | dn42 路由源授权（详见下文路由过滤） | — |
| **dn42 Route Origin Community** | dn42 路由源团体 | — |

---

## 默认扩展

- **Multiprotocol BGP** 为首选协议，**默认启用**。如果您无法支持多协议 BGP 会话，请在对等互联时提前告知。
- **Extended Next Hop** 和 **Extended Messages** 支持均**默认启用**。

---

## 路由过滤

本网络对**所有接收和导出的路由**应用严格的 **Route Origin Authorisation（ROA）** 过滤。

### 过滤规则

- 任何在 dn42 注册表中**没有对应 `route` / `route6` 对象**的广播路由，**将被直接丢弃**~~（吃掉）~~。
- ROA 通过 RPKI 实现，使用 `dn42regsrv` 和 `stayrtr` 进行数据更新。

> dn42 ROA 数据作为公共服务提供，详情请参阅 [DN42 服务页面](https://wiki.dn42/services)。

---

### 通用允许前缀

以下前缀范围内的路由允许被接收和导出：

#### IPv4

- `172.20.0.0/14` 及以上（即 `/14`、`/15`、`/16` … `/32`）
- `10.0.0.0/8` 及以上（即 `/8`、`/9`、`/10` … `/32`）

#### IPv6

- `fd00::/8` 且前缀长度在 `/44` 至 `/64` 之间

> *超出上述范围的路由将不会被接收或导出（~~会被吃掉~~）。*

## 对等流程

1. **确认信息**：检查您是否满足上述对等要求。
2. **提交申请**：通过 IRC 或电子邮件提交对等申请，需包含以下信息：
   - 您的 ASN
   - 您想要宣告的 IP 地址段
   - 您的 WireGuard 公钥 或 GRE 隧道参数
   - 您偏好的 BGP 会话配置（如多跳数、Keepalive 时间等）
3. **协商与配置**：双方协商隧道参数并完成配置。
4. **测试验证**：确认 BGP 会话建立，并验证路由宣告正常。

## 常见问题

### 是否支持多跳 BGP（Multihop）？

支持。请提前告知您偏好的 TTL 和 Keepalive 配置。

### 是否可以对等 IPv6-only 节点？

支持。我们已启用 IPv6 双栈支持，并支持扩展下一跳（Extended Next-Hop）。
